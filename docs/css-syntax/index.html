Die Syntax von CSS

Deklarationen#

CSS-Deklarationen bestehen stets aus einer Eigenschaft (wie zum Beispiel background-color für die Hintergrundfarbe, font-family für die Schriftart oder border-width für die Rahmenbreite) und einem Wert (z.B. red, Arial oder 1px).

Eine CSS-Deklaration besteht aus der Eigenschaft, gefolgt von einem Doppelpunkt und dem Wert:

background-color: red

Es gibt über 100 CSS-Eigenschaften.

Deklarationsblöcke#

Deklarationen werden in Blöcken zusammengefasst. Ein Block wird von geschwungenen Klammern umgeben, besteht dann aus einer oder mehreren Deklarationen, jeweils gefolgt von einem Strichpunkt:

{
background-color: red;
font-family: Helvetica, Arial, Sans-Serif;
border-width: 1px;
}

CSS-Regelsätze#

Ein CSS-Regelsatz definiert nun, auf welche Elemente ein Deklarationsblock angewendet werden soll. Ein Regelsatz besteht daher aus einem Selektor, der festlegt, worauf sich der Regelsatz bezieht, und einem Deklarationsblock, der definiert, welche Regeln angewendet werden sollen.

p.important, div.statement {
background-color: red;
font-family: Helvetica, Arial, Sans-Serif;
border-width: 1px;
}

Dieser Block definiert, dass alle p-Elemente, deren class-Attribut den Wert important hat rot, mit Schriftart Helvetica (alternativ mit Arial bzw. einer generischen serifenlosen Schrift) und einem 1 Pixel breiten Rahmen dargestellt werden sollen. Dieselben Regeln gelten für alle div-Elemente, deren class-Attribut den Wert statement haben.

Selektoren#

Mit Selektoren kann festgelegt werden, auf welche Elemente CSS-Regelsätze angewendet werden sollen. Wie vorhin beschrieben, wird mit Hilfe dieser Selektoren entschieden, welche Knoten im DOM mit welchen Stilinformationen verknüpft werden:

Universeller Selektor#

Der Universelle Selektor * selektiert alle Elemente des Dokuments. z.B.:

* {
font-family: 'Comic Sans MS'
}

Typselektoren#

Typselektoren selektieren alle Elemente eines gewissen Typs, also z.B. alle h1-Elemente, alle p-Elemente, ... . Sie bestehen aus dem Elementnamen:

h1 {
font-size: 16pt;
font-weight: 200;
}

Klassenselektoren#

Klassenselektoren selektieren alle Elemente gemäß dem Wert ihres class-Attributs. Sie bestehen aus einem . gefolgt vom Klassennamen. Oft werden sie auch kombiniert mit Typselektoren, die richtige Syntax dafür ist elementname.classname:

.warning {
color: orange;
}
div.error {
color: red;
}

ID-Selektoren#

ID-Selektoren selektieren ein Element gemäß dem Wert des id-Attributs. ID-Attribute sollten innerhalb eines Dokuments stets eindeutig sein. Diese Selektoren bestehen aus einem # gefolgt vom ID-Wert. Auch sie können mit Elementnamen kombiniert werden, auch wenn das ob der Eindeutigkeit des ID-Selektors nicht notwendig ist:

#title {
font-size: larger;
}

Pseudoklassen#

Mit Pseudoklassen können Elemente abhängig von ihrem Zustand selektiert werden. Beispielsweise bei Links können Links, die dieder Benutzerin bereits besucht hat, anders dargestellt werden als solche, die noch nicht besucht wurden:

a:visited {
color: slategray;
}

Eine ausführliche Dokumentation der Pseudoklassen finden Sie bei Mozilla.

Pseudoelemente#

Mit Pseudoelementen können gewisse Teile einzelner Elemente selektiert werden, z.B. die erste Zeile oder ein fiktives Element vor oder nach einem Element.

a::after {
content: '>';
}
p::first-line {
padding-left: 2rem;
}

Eine ausführliche Dokumentation der Pseudoelemente finden Sie bei Mozilla.

Attribut-Selektoren#

Mit Attribut-Selektoren können Elemente abhängig von gewissen Attributwerten selektiert werden, so z.B.:

/* selektiert alle a-Elemente mit einem title-Attribut */
a[title] {
background-color: gray;
}
/* selektiert alle a-Elemente, die auf www.campus02.at verlinken */
a[title="https://www.campus02.at"] {
font-size: 2rem;
}

Eine ausführliche Dokumentation der Attribut-Selektoren finden Sie bei Mozilla.

Selektoren kombinieren#

Mehrere Selektoren können auch kombiniert werden:

  • Selektoren können gruppiert werden, indem sie durch Beistriche getrennt werden, z.B.: div, span.warning, p { ... }
  • Kind-Kombinationen können verwendet werden, wenn Regeln auf direkte Kinder anderer Elemente angewendet werden sollen. Sollen z.B. alle li-Elemente in ul-Elementen selektiert werden, dann erfolgt dies mithilfe eines >: ul > li
  • Nachkommen-Kombinationen können verwendet werden, wenn Regeln auf alle Nachkommen (also Kinder, deren Kinder & Kindeskinder, ...) angewendet werden sollen. Dazu gibt man einfach ein Leerzeichen zwischen den Selektoren an: nav a selektiert alle a-Elemente, die sich irgendwo in einem nav-Element befinden
  • Geschwister-Kombinationen können verwendet werden, wenn Regeln auf nachfolgende Geschwister angewendet werden sollen (also Elemente, die dieselben Eltern im DOM-Baum haben). Der Operator für diese Kombinationen ist die ~: p ~ span selektiert alle span-Elemente, die auf ein p-Element folgen und die gleichen Eltern haben.
  • Nachbars-Geschwister-Kombinationen können verwendet werden, wenn Regeln auf direkt nachfolgende Geschwister angewendet werden sollen, der Operator dafür ist das +: h2 + p selektiert alle p-Elemente, die direkt auf ein h2-Element folgen.

At-Regeln#

Zusätzlich zu den CSS-Regelsätzen gibt es noch At-Regeln, die wichtigsten sind:

  • @charset definiert den verwendeten Zeichensatz, z.B. @charset "utf-8;"
  • @import zum Importieren externer Stylesheets, z.B. @import "custom.css"; oder import url('landscape.css') screen and (orientation:landscape);
  • @font-face zum Importieren von Schriftarten, z.B. @font-face { font-family: MyHelvetica; src: local("Helvetica Neue Bold"), local("HelveticaNeue-Bold"), url(MgOpenModernaBold.ttf); font-weight: bold; }
  • @keyframes für Animationen, die im Rahmen dieses Skriptums nicht behandelt werden. Mehr zu Animationen finden Sie u.a. bei Mozilla
  • @page um spezielle Regeln für Ausdrucke zu definieren, z.B. @page :first { margin: 2cm; }

Weiters gibt es die etwas komplexeren Regeln @media und @supports:

@media#

Mit der @media-Regel können ausgabemedienabhängige Bedingungen definiert werden. Nur, wenn diese Bedingungen erfüllt werden, werden auch die zugehörigen Regelsätze angewendet. Soll z.B. die Zeilenhöhe größer gewählt werden, wenn ein Benutzer eine Seite auf einem Bildschirm mit über 1200px Breite ansieht, kann das so definiert werden:

@media (width >= 1200px) {
body { line-height: 1.4; }
}
@media (800px <= width < 1200px) {
body { line-height: 1.2; }
}

Eine umfassende Beschreibung der media-Queries gibt es bei Mozilla.

@supports#

Mit der @supports-Regel können Regelsätze abhängig von vom Browser unterstützten Features angewendet werden. Möchte man zum Beispiel in Browsern, die das unterstützen, ein Grid-Layout verwenden, in allen anderen ein Flex-Layout (auf diese Art von Layouts werden wir noch eingehen), kann das so formuliert werden:

.layout {
display: flex;
flex-wrap: wrap;
}
@supports (display: grid) {
.layout {
display: grid;
grid-template-columns: repeat(auto-fill, minmax(100px, 1fr));
grid-gap: 2rem;
}
}

Nachdem alle modernen Browser ein weitgehend gleiches Set von Features unterstützen, hat die @supports-Regel eine untergeordnete Bedeutung. Eine umfassende Beschreibung der supports-Queries gibt es bei Mozilla.

Externes, internes und inline CSS#

Es gibt drei unterschiedliche Möglichkeiten, CSS-Regeln zu definieren:

  • Externe CSS-Regeln werden in einer separaten CSS-Datei definiert. Diese Dateien werden im Header mit dem link-Element eingebunden
  • Interne CSS-Regeln werden direkt im HTML-Head mit dem style-Tag angegeben
  • Inline-CSS-Regeln, die bei beliebigen HTML-Elementen im Body im style-Attribut angegeben werden. Inline-CSS-Regeln enthalten Deklarationsblöcke ohne die geschwungenen Klammern
<!DOCTYPE html>
<html>
<head>
<link rel="stylesheet" href="styles.css"> <!-- externes Stylesheet styles.css -->
<!-- hier kommt internes CSS -->
<style>
p {
font-weight: bold;
}
</style>
</head>
<body>
<h1>Hallo Welt</h1>
<p>Ein Absatz ohne gesonderte Stilinformationen.</p>
<p style="font-style: italic;">Das ist inline-CSS!</p>
</body>
</html>

Prävalenz von CSS-Regeln#

Wie gerade beschrieben gibt es mehrere Möglichkeiten, CSS-Regeln zu definieren. Welche CSS-Regeln haben aber nun Vorrang? Angenommen, Sie haben folgende HTML-Datei:

<!DOCTYPE html>
<html>
<head>
<link rel="stylesheet" href="styles.css"> <!-- externes CSS enthält Regel h1 { color: red; } -->
<style>
h1 {
color: blue;
}
</style>
</head>
<body>
<h1 style="color: green;">Hallo Welt</h1>
</body>
</html>

Welche Farbe hat dann die Überschrift "Hallo Welt"? In diesem Fall hat die Inline-Regel Vorrang, die Überschrift wird also grün sein. Ohne die Inline-Regel wäre die Überschrift blau, denn die später im Header definierte Regel hat Vorrang gegenüber davor definierter Regeln. Die Vorrangregeln für CSS-Regeln sind also:

  1. Inline-Regeln
  2. Im Header definierte Regeln, wobei später im Header angeführte Regeln Vorrang gegenüber früher angeführten Regeln haben und spezifische Regeln Vorrang haben gegenüber allgemeinen Regeln.
  3. Browser-Standard-Werte

Zum Aushebeln dieses Regelwerks gibt es die Möglichkeit, CSS-Regeln mit !important zu markieren. Solche Regeln haben immer Vorrang. !important sollte aber sehr sparsam eingesetzt werden, spezifischere Regeln und eine nachvollziehbare Reihenfolge der Styles sind stets zu bevorzugen:

div.warning {
background-color: red !important;
}
