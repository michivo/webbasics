**Cascading Style Sheets** (CSS) werden verwendet, um die Darstellung eines HTML-Dokuments zu beschreiben. Während also 
*Inhalt* und *Struktur* mit HTML beschrieben werden, dient CSS der Beschreibung der *Darstellung* bzw. *Präsentation*.
Gemeinsam mit HTML und JavaScript ist CSS der wichtigste Standard bei der Erstellung von Webseiten.

Style Sheets ganz allgemein beinhalten Informationen darüber, wie Inhalte dargestellt werden sollen. Bei 
*kaskadierenden* Stylesheets ist es nun so, dass es mehrere Quellen geben kann, woher Angaben zur
Darstellung kommen und es ein Regelwerk gibt, in welcher Reihenfolge diese Angaben ausgewertet werden,
dazu aber später mehr.

Was kann man nun mit CSS "stylen"?
- Das Schriftbild: Welche Farbe, Schriftart, Schriftgröße, ... hat Text?
- Rahmen, Hintergrundfarben, ... von Elementen
- Die Positionierung von Elementen

## Geschichte

Anfangs wurden Informationen zur Darstellung direkt in HTML angeführt. Das hat aber einige Nachteile: Es wird vor allem
bei großen Seiten mühsam, einen konsistenten Stil zu erreichen und beizubehalten, Änderungen an diesem Stil sind schwer
umzusetzen, und es ist aufwändig, mehrere Versionen einer Seite (z.B. für Mobilgeräte und Desktop, für Darstellung am Bildschirm
und zum Ausdruck, barrierefreie Darstellung, ...) zu warten. Deshalb gab es schon recht bald nach Veröffentlichung des
HTML-Standards Bestrebungen, die Beschreibung des *Inhalts* von jener der *Darstellung* zu trennen, und zwar mit Style Sheets. 
1994 wurde CSS von Håkon Wium Lie, der mit Tim Berners Lee am CERN zusammenarbeitete, und Bert Bos entworfen und 1996 als Standard 
vom W3C veröffentlicht. 

CSS wird auch nach wie vor vom W3C gepflegt und erweitert. Ähnlich wie bei HTML hat sich eine Art Living Standard etabliert, die
CSS-Spezifikation besteht mittlerweile aus vielen separaten Modulen, die unabhängig voneinander weiterentwickelt werden. Der letzte
offizielle Standard war CSS3, CSS4 wurde nie offiziell verabschiedet.

## Funktionsweise

Mit CSS werden Styling-Regeln definiert, die dann vom Browser angewandt werden. Der Browser liest ein HTML-Dokument ein, erzeugt
daraus ein Document Objet Model. Der Browser lädt anschließend weitere Ressourcen, die für die Darstellung des HTML-Dokuments
benötigt werden, so unter anderem Stylesheets. Die Stylesheets werden vom Browser eingelesen und verarbeitet und verknüpft 
anschließend die Style-Informationen aus den Stylesheets mit dem DOM. Erst dann wird die Seite im Browser dargestellt.
